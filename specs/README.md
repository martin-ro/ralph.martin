# Specifications

This directory contains all feature and system specifications. Each spec defines requirements, architecture, and acceptance criteria for a topic of concern.

## Spec Index

| Spec | Status | Description |
|------|--------|-------------|
| [filament-panel.md](filament-panel.md) | Ready | Filament v5 panel setup at root path with login/password reset |

## Status Legend

- **Draft** - Spec is being written, not ready for implementation
- **Ready** - Spec is approved and ready for implementation
- **In Progress** - Implementation has started
- **Implemented** - Feature is complete and tested

## Creating New Specs

1. Create a new file: `specs/<feature-name>.md`
2. Add entry to this index table
3. Follow the spec template structure:
   - Overview (purpose, goals, non-goals)
   - Requirements (functional, non-functional)
   - Architecture (if applicable)
   - Acceptance Criteria
   - Dependencies (other specs or external systems)
